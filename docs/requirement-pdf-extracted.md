## 项目主题：构建一个"轻量级分布式任务调度系统 (Mini-Scheduler)"

### 一、 业务背景
为计算集群构建一个可视化调度系统。要求后台稳定，前端界面"炫酷"且"实时"。

### 二、 后端核心要求

1. **节点注册 (Node Registration)：**
   - Worker 启动后必须自动向 Master 注册。
   - 汇报内容：CPU 核数和内存大小。

2. **任务提交 (Task Submission)：**
   - 通过 **REST API** 提交。
   - 任务参数：`command`（执行命令）、`cpu_required`、`mem_required`。

3. **智能调度 (Bin Packing)：**
   - **算法逻辑：** Master 必须根据 Worker **剩余资源**进行分配。
   - **示例场景：** 若 Worker A 有 4核/8G 资源，提交两个各需 2核/4G 的任务，应能同时调度至该 Worker。

4. **任务状态追踪 (Status Tracking)：**
   - 必须能实时查询任务状态。
   - 状态定义：`Pending`（等待中）, `Running`（运行中）, `Success`（成功）, `Failed`（失败）。

### 三、 前端核心要求 (技术栈：Vue 3 + AI)
必须使用 **Vue.js** (推荐 **Vue 3 Composition API**)。

- **功能 A：集群资源热力图 (Cluster Heatmap)**
  - **可视化：** 以卡片形式展示所有 Worker 节点，动态显示 CPU/内存实时占用率（进度条或仪表盘）。
  - **实时性：** 数据需动态跳动，严禁手动刷新。必须使用 **WebSocket** 或**短轮询 (Short Polling)**。
  - **状态感知：** 若 Worker 心跳丢失（挂掉），前端卡片需变灰或显示"OFFLINE"。

- **功能 B：任务日志流 (Live Log Stream)**
  - **交互：** 点击运行中的任务，弹出模态框 (Modal) 或侧边栏。
  - **核心难点：** 实时通过 **WebSocket** 接收并展示任务的 `stdout` 日志。
  - **功能细节：** 必须实现"**自动滚动到底部**"的功能。

- **功能 C：AI 辅助交互设计**
  - **美观度：** 可使用 Element Plus, AntDV 或 Tailwind CSS。
  - **考核点：** 使用 AI 生成合理的 UI 布局，确保界面符合人类直觉。
