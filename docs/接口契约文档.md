## 0. 全局约定

### 0.1 命名风格

| 层 | 风格 | 示例 |
|----|------|------|
| REST API URL | kebab-case | `/api/tasks`, `/internal/heartbeat` |
| JSON 字段（前后端通信） | **snake_case** | `task_id`, `cpu_required`, `worker_id` |
| Python 后端变量 | snake_case | `remaining_cpu`, `last_heartbeat` |
| TypeScript 前端变量 | camelCase | `taskId`, `cpuRequired`（仅前端内部使用） |
| 前端 ↔ 后端 JSON 传输 | **统一 snake_case** | 前端发送和接收 JSON 均使用 snake_case |

> **重要：** 前后端 JSON 交互统一使用 **snake_case**。前端内部可使用 camelCase，但在 API 调用层做转换。

### 0.2 通用响应格式

**成功响应：**
```json
{
  "code": 0,
  "message": "success",
  "data": { ... }
}
```

**错误响应：**
```json
{
  "code": 40001,
  "message": "Task not found",
  "data": null
}
```

### 0.3 状态枚举

#### 任务状态 (TaskStatus)

| 值 | 说明 | 是否终态 |
|----|------|----------|
| `pending` | 等待调度（资源不足或无可用 Worker） | 否 |
| `running` | Worker 正在执行 | 否 |
| `success` | 执行成功（退出码 = 0） | 是 |
| `failed` | 执行失败（退出码 ≠ 0 或 Worker 下线） | 是 |

#### Worker 状态 (WorkerStatus)

| 值 | 说明 |
|----|------|
| `online` | 已注册且心跳正常（15 秒内有心跳） |
| `offline` | 心跳超时（超过 15 秒无心跳） |

### 0.4 错误码

| 错误码 | HTTP 状态码 | 说明 |
|--------|------------|------|
| 0 | 200 | 成功 |
| 40001 | 404 | 任务不存在 |
| 40002 | 404 | Worker 不存在 |
| 40003 | 400 | 请求参数无效（缺少必填字段或类型错误） |
| 40004 | 400 | CPU 或内存请求值超出所有 Worker 最大容量 |
| 40005 | 409 | Worker ID 已注册（重复注册） |
| 50001 | 500 | 调度器内部错误 |
| 50002 | 500 | Worker 通信失败（无法下发任务） |

### 0.5 公共数据类型

#### TaskInfo

```json
{
  "task_id": "task-uuid-1234",
  "command": "python train.py --epochs 10",
  "cpu_required": 2,
  "mem_required": 4,
  "status": "running",

  "created_at": "2026-03-13T10:00:00Z",
  "started_at": "2026-03-13T10:00:05Z",
  "finished_at": null
}
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|

| command | string | 是 | 执行命令 |
| cpu_required | int | 是 | 需要的 CPU 核数 |

| status | string | 是 | 枚举：pending / running / success / failed |
| worker_id | string \| null | 是 | 分配到的 Worker ID，Pending 时为 null |
| created_at | string | 是 | ISO 8601 创建时间 |
| started_at | string \| null | 是 | ISO 8601 开始执行时间，未开始为 null |
| finished_at | string \| null | 是 | ISO 8601 完成时间，未完成为 null |

| host | string | 是 | Worker IP 地址 |

## 1. REST API — 前端调用（/api/*）

### 1.1 POST /api/tasks — 提交任务

**请求：**
```http
POST /api/tasks
Content-Type: application/json

{
  "command": "python train.py --epochs 10",
  "cpu_required": 2,
  "mem_required": 4
}
```

| 字段 | 类型 | 必填 | 校验规则 |
|------|------|------|----------|
| command | string | 是 | 非空，最大 1024 字符 |
| cpu_required | int | 是 | ≥ 1 |
| mem_required | int | 是 | ≥ 1，单位 GB |

**成功响应 (200)：**
```json
{
  "code": 0,
  "message": "success",
  "data": {
    "task_id": "task-uuid-1234",
    "command": "python train.py --epochs 10",
    "cpu_required": 2,
    "mem_required": 4,
    "status": "pending",
    "worker_id": null,
    "created_at": "2026-03-13T10:00:00Z",
    "started_at": null,
    "finished_at": null
  }
}
```

**说明：**
- Master 生成 `task_id`（UUID）
- 立即返回 `pending` 状态，调度异步进行
- 若调度成功（找到合适 Worker），状态异步变为 `running`
- 若无可用资源，保持 `pending` 直到资源释放

**错误响应：**
```json
{
  "code": 40003,
  "message": "Invalid parameter: cpu_required must be >= 1",
  "data": null
}
```

## 2. REST API — Worker 内部通信（/internal/*）

> 这些接口仅供 Worker 进程调用，不暴露给前端。

### 2.1 POST /internal/register — Worker 注册

**请求：**
```http
POST /internal/register
Content-Type: application/json

{
  "worker_id": "worker-01",
  "host": "192.168.1.10",
  "port": 8001,
  "total_cpu": 4,
  "total_mem": 8
}
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| worker_id | string | 是 | Worker 唯一标识 |
| host | string | 是 | Worker IP 地址 |
| port | int | 是 | Worker 服务端口 |
| total_cpu | int | 是 | CPU 总核数 |
| total_mem | int | 是 | 内存总量（GB） |

**成功响应 (200)：**
```json
{
  "code": 0,
  "message": "success",
  "data": {
    "worker_id": "worker-01",
    "registered": true
  }
}
```

**说明：**
- Worker 启动时调用，注册到 Master
- 若 worker_id 已存在且为 OFFLINE，允许重新注册（恢复上线）
- 若 worker_id 已存在且为 ONLINE，更新信息（幂等注册）
- 注册成功后 Master 初始化 `used_cpu=0, used_mem=0`

---

### 2.2 POST /internal/heartbeat — Worker 心跳

**请求：**
```http
POST /internal/heartbeat
Content-Type: application/json

{
  "worker_id": "worker-01"
}
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| worker_id | string | 是 | Worker 唯一标识 |

**成功响应 (200)：**
```json
{
  "code": 0,
  "message": "success",
  "data": {
    "worker_id": "worker-01",
    "exist": true
  }
}
```

**说明：**
- Worker 每 **5 秒** 发送一次心跳
- Master 更新 `last_heartbeat` 时间戳
- 若 `exist: false`，说明 Master 不认识该 Worker（可能重启过），Worker 需重新注册
- Master 后台每 **5 秒** 扫描一次，超过 **15 秒** 未心跳的 Worker 标记为 OFFLINE

---

### 2.3 POST /internal/task/{task_id}/status — Worker 上报任务状态

**请求：**
```http
POST /internal/task/task-uuid-1234/status
Content-Type: application/json

{
  "worker_id": "worker-01",
  "status": "running"
}
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| worker_id | string | 是 | 上报的 Worker ID |
| status | string | 是 | 枚举：running / success / failed |

**成功响应 (200)：**
```json
{
  "code": 0,
  "message": "success",
  "data": {
    "task_id": "task-uuid-1234",
    "status": "running"
  }
}
```

**说明：**
- Worker 在以下时机调用：
  - 开始执行任务 → `status: "running"`
  - 任务成功完成 → `status: "success"`
  - 任务执行失败 → `status: "failed"`
- 当状态为 success/failed 时，Master 释放该任务占用的 CPU/内存资源
- 资源释放后触发 Pending 任务重调度

---

### 2.4 POST /internal/task/{task_id}/log — Worker 上报日志

**请求：**
```http
POST /internal/task/task-uuid-1234/log
Content-Type: application/json

{
  "worker_id": "worker-01",
  "line_no": 42,
  "content": "Epoch 5/10, loss=0.3421"
}
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| worker_id | string | 是 | 上报的 Worker ID |
| line_no | int | 是 | 日志行号（从 1 开始递增） |
| content | string | 是 | 日志内容（一行 stdout） |

**成功响应 (200)：**
```json
{
  "code": 0,
  "message": "success",
  "data": null
}
```

**说明：**
- Worker 逐行读取子进程 stdout，每行发送一次
- Master 接收后：
  1. 存入 RingBuffer（最近 1000 行/任务）
  2. 广播给订阅该 task_id 的所有 WebSocket 客户端
- 为减少 HTTP 开销，可选优化：Worker 本地缓冲 50ms 批量发送（MVP 不需要）

## 3. Worker API — Master 调用（Worker 侧端点）

### 3.1 POST /execute — Master 下发任务

**请求（Master → Worker）：**
```http
POST http://worker-01:8001/execute
Content-Type: application/json

{
  "task_id": "task-uuid-1234",
  "command": "python train.py --epochs 10",
  "cpu_required": 2,
  "mem_required": 4
}
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| task_id | string | 是 | 任务唯一 ID |
| command | string | 是 | 待执行的命令 |
| cpu_required | int | 是 | 分配的 CPU（用于 Worker 本地记录） |
| mem_required | int | 是 | 分配的内存（用于 Worker 本地记录） |

**成功响应 (200)：**
```json
{
  "code": 0,
  "message": "success",
  "data": {
    "task_id": "task-uuid-1234",
    "accepted": true
  }
}
```

**说明：**
- Worker 收到后立即返回 `accepted: true`，然后异步启动子进程
- Worker 通过 `asyncio.create_subprocess_shell` 执行 command
- Worker 异步上报状态（running → success/failed）和日志（逐行）

## 4. WebSocket 协议 — Master → 前端

### 4.1 WS /ws/cluster — 集群状态广播

**连接：**
```
ws://master-host:8000/ws/cluster
```

**推送方向：** Master → 前端（单向推送）  
**推送频率：** 每 **1 秒** 推送一次全量数据  

#### 数据帧格式

**集群状态帧：**
```json
{
  "type": "cluster_state",
  "timestamp": "2026-03-13T10:05:30Z",
  "data": {
    "workers": [
      {
        "worker_id": "worker-01",
        "total_cpu": 4,
        "total_mem": 8,
        "used_cpu": 2,
        "used_mem": 4,
        "status": "online",
        "task_count": 1
      },
      {
        "worker_id": "worker-02",
        "total_cpu": 8,
        "total_mem": 16,
        "used_cpu": 0,
        "used_mem": 0,
        "status": "offline",
        "task_count": 0
      }
    ],
    "tasks_summary": {
      "pending": 3,
      "running": 5,
      "success": 12,
      "failed": 1
    }
  }
}
```

#### 消息类型枚举

| type | 说明 | 方向 |
|------|------|------|
| `cluster_state` | 全量集群状态 | Master → 前端 |
| `ping` | 客户端心跳 | 前端 → Master |
| `pong` | 心跳响应 | Master → 前端 |

**连接管理：**
- 前端连接后立即收到第一帧 `cluster_state`
- 之后每秒推送一次
- 前端应每 30 秒发送 `ping`，Master 回复 `pong`
- 60 秒无 `ping` → Master 关闭连接

---

### 4.2 WS /ws/logs/{task_id} — 任务日志流

**连接：**
```
ws://master-host:8000/ws/logs/task-uuid-1234
```

**推送方向：** Master → 前端（单向推送）  
**推送频率：** 即时（Worker 上报一行，立刻推送）  

#### 数据帧格式

**连接确认帧：**
```json
{
  "type": "connected",
  "task_id": "task-uuid-1234",
  "timestamp": "2026-03-13T10:05:30Z"
}
```

**历史日志帧（连接时回放）：**
```json
{
  "type": "history",
  "task_id": "task-uuid-1234",
  "lines": [
    {"line_no": 1, "content": "Starting training...", "timestamp": "2026-03-13T10:00:05Z"},
    {"line_no": 2, "content": "Epoch 1/10, loss=0.8934", "timestamp": "2026-03-13T10:00:10Z"}
  ]
}
```

**实时日志帧：**
```json
{
  "type": "log",
  "task_id": "task-uuid-1234",
  "line_no": 42,
  "content": "Epoch 5/10, loss=0.3421",
  "timestamp": "2026-03-13T10:02:30Z"
}
```

**任务完成帧：**
```json
{
  "type": "task_completed",
  "task_id": "task-uuid-1234",
  "status": "success",
  "exit_code": 0,
  "timestamp": "2026-03-13T10:10:00Z"
}
```

#### 消息类型枚举

| type | 说明 | 方向 |
|------|------|------|
| `connected` | 连接确认 | Master → 前端 |
| `history` | 历史日志回放 | Master → 前端 |
| `log` | 实时日志行 | Master → 前端 |
| `task_completed` | 任务结束通知 | Master → 前端 |
| `ping` | 客户端心跳 | 前端 → Master |
| `pong` | 心跳响应 | Master → 前端 |

**连接管理：**
- 连接时先发送 `connected` 帧
- 然后发送 `history` 帧（RingBuffer 中的已有日志）
- 之后实时推送 `log` 帧
- 任务完成后发送 `task_completed` 帧
- 前端关闭模态框 → 断开 WebSocket → Master 清理该订阅
